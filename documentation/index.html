This site is project for web publishing. It is created as static website using jekyll. The site is written in HTML, enhanced with jekyll additions. The look of the page is created with CSS.

I used 3 layouts: default - primary layout with header, footer and content place, list to show our collection of projects and project for showing information about the project with code examples. You can click on the links or you can find them directly with urls (for example ~{{ site.baseurl }}{% link _projects/AVL-tree.md %})

These layout uses variables from corresponding sites, where are defined. For example project variables as date, title, subject, semester, language and teacher. It is clever, that I don't need to write just html, but can use these variables and jekyll creates the full content.

This projects are created as collection. In jekyll I just created corresponding folder with my projects in markdown and then listed them easily with the for cycle.

Jekyll provides a number of useful liquid additions as filters: number-of-words, date-to-long, to-integer and tags: highlight, baseurl.

In my site is youtube plugin also used to showing videos directly here. I just had to add this lines in my Gemfile and refresh the site:

{% highlight ruby %} group :jekyll_plugins do gem "jekyll-youtube" end {% endhighlight %}
